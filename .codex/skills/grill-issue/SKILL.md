---
name: grill-issue
description: issueの内容を読み取り実装の計画を立てる。議論が不十分な場合議論を行う。
---

この計画について、相互理解に到達するまで、あらゆる側面を徹底的にインタビューしてください。  
設計ツリーの各分岐を順番にたどり、意思決定間の依存関係を一つずつ解消していってください。  
各質問に対して、推奨される回答例も提示してください。

質問は必ず一度に一つだけしてください。

もし質問の答えをコードベースの調査によって得られるなら、ユーザーに聞くのではなくコードベースを調査してください。

議論内容として以下のコマンドでIssueの内容を取得してください:

```
gh issue view <issue-number>
```
