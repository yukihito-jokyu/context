# AGENTS.md

このファイルは、`context/cli` で作業する AI Agent 向けの最優先エントリポイントです。作業を始める前に必ず読み、このプロジェクト固有の方針に従ってください。

## 1. このプロジェクトの位置づけ

- このディレクトリは、Go 製 `context` CLI の実装場所です。
- 現時点では `context deploy` の初期実装があり、今後 `deploy` 系コマンドの拡張を想定しています。
- このリポジトリは AI Agent 向けコンテキスト管理のためのリポジトリであり、CLI 単体で完結するのではなく、上位の運用方針と整合していることが重要です。

## 2. 参照順序

作業時は、以下をこの順序で確認してください。

1. [../AGENTS.md](../AGENTS.md)
   - このリポジトリ全体の管理方針と、正本管理の前提を確認します。
2. [README.md](README.md)
   - `cli/` ディレクトリの現況と、想定しているコマンド範囲を確認します。
3. 必要に応じて `../docs/` や `../projects/context/` 配下
   - 要件、設計、運用方針、将来の実装意図を確認します。

## 3. 実装時の基本方針

- まだ初期段階であることを前提に、既存コードに合わせるだけでなく、リポジトリ全体の管理方針に沿って構造を提案・実装してください。
- CLI の具体的な利用手順や運用手順は、このファイルに肥大化させず、必要なら Skill・ドキュメント・CLI ヘルプへ分離してください。
- 重複した仕様記述を避け、すでに上位ドキュメントにある内容は参照で済ませてください。
- 将来の `deploy` 系コマンド拡張を見越し、責務分離しやすい構成を優先してください。

## 4. 変更時の期待

- 新しいコマンド構造、設定形式、永続化形式など、将来の保守性に影響する決定をした場合は、その理由をドキュメントに残してください。
- 実装に伴って `README.md` の説明が古くなる場合は、コード変更と同じ作業単位で更新してください。
- 上位の `AGENTS.md` や関連ドキュメントと矛盾する場合は、このファイルだけで判断せず、正本側の整理を優先してください。

## 5. 禁止事項

- `cli/AGENTS.md` を独自ルールの正本として扱わないでください。
- 親ディレクトリの方針と食い違うローカル運用ルールを、このファイルに追加しないでください。
- 実装詳細の手順書をここに蓄積し続けないでください。
