# 深いモジュール

「ソフトウェア設計の哲学」より：

**深いモジュール** = 小さなインターフェース + 多くの実装

```
┌─────────────────────┐
│   Small Interface   │  ← 少数のメソッド、シンプルなパラメータ
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 複雑なロジックが隠されている
│                     │
│                     │
└─────────────────────┘
```

**浅いモジュール** = 大きなインターフェース + 少しの実装（避けるべき）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 多数のメソッド、複雑なパラメータ
├─────────────────────────────────┤
│  Thin Implementation            │  ← 単なるパススルー
└─────────────────────────────────┘
```

インターフェースを設計する際、問う：

- メソッドの数を減らせるか？
- パラメータをシンプルにできるか？
- 内部にもっと複雑さを隠せるか？
